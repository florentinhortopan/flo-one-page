Leading accessibility improvements and component adoption across Airbnb's global platform

The Problem

Airbnb's Design Language System was facing critical challenges that impacted both the user experience and designer/developer efficiency:

Key Challenges

  • Multiple product surfaces using outdated components (less than 60% DLS adoption)
  • Growing accessibility backlog creating barriers for users with disabilities
  • No standardized way to add alt-text to images across the platform
  • Fragmented component libraries across different design tools
  • Designers and developers lacking clear guidance on implementing accessible patterns

The lack of alt-text functionality was particularly problematic—screen reader users couldn't access image content, violating WCAG accessibility standards and Airbnb's commitment to inclusive design.

The Solution: Systematic Accessibility Improvements

I led a comprehensive initiative to address these challenges, with a primary focus on developing an alt-text feature that could be implemented across all Airbnb surfaces. This work involved multiple iterations, stakeholder alignment, and cross-functional collaboration.

Design Process & Audit

I conducted a thorough audit of existing components and interfaces, documenting accessibility gaps and component usage patterns across different product areas.

Comprehensive design audit: All parts reviewed and annotated for accessibility improvements

Alt-text Feature: Iterative Development

The alt-text feature went through three major iterations, each addressing different stakeholder needs and technical constraints while maintaining usability and accessibility standards.

Phase I: Low Hanging Fruit

Initial Implementation

The first phase focused on getting a basic alt-text feature into production quickly. Since no alt-text input field existed on desktop or mobile interfaces, I worked with developers to implement a simple text area as a discreet label.

Phase II: Enhanced Discoverability

Second Iteration

Based on user feedback, the second iteration improved feature discoverability and provided better contextual guidance for hosts uploading photos.

Phase III: Optimized Experience

Final Iteration

The final iteration refined the interaction patterns, improved the visual design, and ensured consistency across all photo upload touchpoints.

Design System Consolidation

Beyond the alt-text feature, I worked on consolidating component libraries and improving DLS documentation to support designers and developers.

System-Level Improvements

  • Standardized component variants across desktop and mobile
  • Created comprehensive accessibility guidelines for each component
  • Established clear processes for designers to contribute to DLS
  • Improved component documentation with code examples
  • Facilitated workshops to increase DLS adoption across product teams
